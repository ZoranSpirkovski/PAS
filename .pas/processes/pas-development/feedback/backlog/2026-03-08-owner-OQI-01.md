Target: process:pas-development
Opportunity: OBSERVED
Behavior: Claude Code's native plan mode does not prioritize invoking /pas-development or /pas when planning PAS framework changes. The dogfooding work in cycle-6 was planned and executed outside the process — the plan was produced in native plan mode rather than through a /pas-development cycle. This means the process's own Discovery/Planning phases are bypassed, losing the multi-agent synthesis and structured feedback collection they provide.
Suggestion: Investigate whether CLAUDE.md or skill descriptions can be tuned to make plan mode recognize when /pas-development should be the entry point for PAS evolution work. Alternatively, consider whether this is a Claude Code platform limitation that needs a different mitigation (e.g., a hook or explicit user habit).
